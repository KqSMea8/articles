Contributing guidelines

Thanks for taking the time to contribute!

While this project absolutely welcomes code changes and optimization suggestions, we'd be truly delighted by content submissions. Anyone seeking a guest author opportunity is encouraged to submit a post as a PR. Topics include, but are not limited to:

  • Web development
  • Project management
  • Agile & Scrum
  • Open source communities
  • Diversity & inclusivity in tech

Formatting your post

To create a new post, create an .md file within the _posts directory, using the following format: year-month-day-title-of-post.md

Here are some resources for properly formatting your post:

  • Markdown
  • Jekyll

Reporting an issue

If you see an issue with the site (or have ideas on how to better style and layout the content) feel free to create an issue as a first step. Reference the issue number in your PR should you want to suggest a fix.
